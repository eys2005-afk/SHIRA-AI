# תשתית הקלטה ואוטומציה - פתיחת תיק בשירה

תיקייה זו מכילה כלים להקלטת תהליך "פתיחת תיק" במערכת שירה (`http://shira2`)
באמצעות Playwright, כצעד ראשון לבניית אוטומציה שתפתח תיקים אוטומטית מתוך
Shira AI.

**הערה חשובה:** מערכת שירה זמינה רק מתוך הרשת הפנימית של בתי הדין (כמו
שמתואר ב-`CLAUDE.md` לגבי `shira_proxy.py`). לכן את ההקלטה עצמה צריך
להריץ ממחשב שמחובר לרשת הזו (תחנת עבודה בבית הדין) ולא מהסביבה שבה נוצרה
תשתית זו. כל מה שצריך כבר מוכן כאן - פשוט מעתיקים את התיקייה הזו למחשב
המתאים ומריצים.

## מה יש כאן

| קובץ | תפקיד |
|---|---|
| `SETUP.bat` | התקנה חד-פעמית: מתקין את Playwright ואת הדפדפן |
| `config.example.json` | תבנית הגדרות - להעתיק ל-`config.json` ולמלא |
| `RECORD_OPEN_CASE.bat` | מריץ הקלטה של תהליך פתיחת תיק |
| `record_open_case.py` | הסקריפט שמפעיל את Playwright Codegen |
| `open_case_automation.py` | תבנית לאוטומציה - אחרי ההקלטה ממלאים כאן את הסלקטורים |
| `requirements.txt` | תלויות Python |
| `recordings/` | כאן נשמרות הקלטות גולמיות (לא נכנס ל-git) |

## שלבי הקמה (במחשב היעד, בתוך רשת בית הדין)

1. ודאו ש-Python 3.10+ מותקן (`python --version`).
2. העתיקו את כל תיקיית `automation/` למחשב היעד (USB / רשת / git clone).
3. הריצו `SETUP.bat` פעם אחת - הוא מתקין את חבילת Playwright ואת דפדפן
   Edge הדרוש לה.
4. ודאו ש-`http://shira2` מוגדר באזור "אינטראנט מקומי" (Local Intranet) ב-
   Internet Options של Windows, כדי ש-NTLM יתבצע אוטומטית - בדיוק כמו
   שקורה היום מול `requests-negotiate-sspi` ב-`shira_proxy.py`.
   (Settings → Internet Options → Security → Local intranet → Sites →
   Advanced → הוסיפו `http://shira2`).
5. העתיקו את `config.example.json` לקובץ חדש בשם `config.json` ומלאו בו
   את `open_case_path` - הנתיב בשירה לדף פתיחת תיק (יש לפתוח את הדף הזה
   בדפדפן רגיל בתוך שירה ולהעתיק את הנתיב מתוך ה-URL, למשל
   `/OpenFile.aspx`).

## הקלטת התהליך

1. הריצו `RECORD_OPEN_CASE.bat`.
2. ייפתח חלון דפדפן וגם חלון Playwright Inspector שמציג את הקוד שנוצר
   בזמן אמת.
3. **בצעו את כל תהליך פתיחת התיק עם נתוני בדיקה/דמה בלבד.** אסור להשתמש
   בת"ז, שם, או כל פרט מזהה אמיתי בזמן ההקלטה - הם יישמרו כטקסט גלוי בתוך
   הקוד שנוצר.
4. בסיום, סגרו את חלון הדפדפן. הקוד שנוצר יישמר אוטומטית תחת
   `recordings/open_case_<תאריך-שעה>.py`.

## המרת ההקלטה לאוטומציה לשימוש חוזר

1. פתחו את הקובץ שנוצר ב-`recordings/`.
2. אתרו את ה-selectors של כל שדה רלוונטי (לדוגמה
   `page.fill("#MainContent_txtTZ", ...)`): ת"ז, שם, סוג תיק, כפתור
   שליחה, והאלמנט שמציג את מספר התיק החדש לאחר השליחה.
3. העתיקו את הסלקטורים האלה **בלבד** (לא את הערכים שהוקלדו בזמן
   ההקלטה!) לתוך `FIELD_SELECTORS` בקובץ `open_case_automation.py`.
4. הריצו `python open_case_automation.py` לבדיקה - הוא ירוץ עם נתוני דמה
   קבועים בקוד ויפתח דפדפן גלוי כדי שתוכלו לוודא שהמילוי האוטומטי עובד
   נכון מקצה לקצה.
5. לאחר שזה עובד בצורה יציבה, אפשר למחוק את קובץ ההקלטה הגולמי מ-
   `recordings/` - הוא לא צריך להישמר כי הוא מכיל מה שהוקלד בזמן ההקלטה.

## שילוב עתידי בתוך shira_proxy.py

לאחר שהאוטומציה ב-`open_case_automation.py` עובדת בצורה יציבה ונבדקה
היטב, אפשר להוסיף ל-`shira_proxy.py` endpoint חדש (לדוגמה
`POST /api/automation/open-case`) שמייבא את `open_case()` ומפעיל אותו עם
הנתונים שמגיעים מהטופס בממשק. זה בכוונה לא חלק מהתשתית הנוכחית, כדי לא
לשלב קוד שעדיין לא נבדק ישירות לתוך השרת הראשי.

## אזהרות

- **לעולם אל תריצו את האוטומציה עם נתונים אמיתיים** לפני שבדקתם את כל
  התהליך עם נתוני דמה והוא עובד בצורה יציבה וצפויה.
- קובצי הקלטה גולמיים (`recordings/*.py`) לא נכנסים ל-git
  (ראו `.gitignore`) כי הם עלולים להכיל מידע שהוקלד בזמן ההקלטה.
- `config.json` גם הוא לא נכנס ל-git, כדי שלא יישמרו בו פרטי סביבה
  ספציפיים למחשב.
